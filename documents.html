Documents API

External integration

The Documents API is the Altafid document locker: it stores uploaded files and their metadata, organises them in folders, and governs what may be stored through tenant-defined document specifications. It tracks compliance, signature, and expiration state per document and supports single and bulk download.

Base URL: https://api.altafid.dev.altafid.net
All requests require standard auth headers — see Authentication and Global Headers.
Files are stored in object storage. Uploads are multipart; the stored file is referenced by fileUrl on the document record. Maximum upload size is 10 MB per request. Deletes are soft — a deleted document can be listed and recovered.

Authentication and headers

All endpoints below require the standard gateway headers. x-tenant-uuid scopes every request to a single tenant; no cross-tenant access is possible.

HeaderRequiredNotes
Authorization: Bearer <token>requiredAccess token.
x-tenant-uuidrequiredTenant context.
x-user-idrequiredCurrent user UUID.
x-user-emailrequiredCurrent user email.
x-user-typerequiredSYSTEM_STAFF, TENANT_STAFF, or PARTNER_STAFF.
x-partner-uuidoptionalRequired for partner staff.

Object model

Documents sit at the bottom of a small hierarchy. Types and categories are platform reference data; specifications are defined per tenant and decide what may be uploaded and how it is governed; folders organise the stored files.

ConceptScopeWhat it is
Document TypePlatform referenceTop-level classification, e.g. Document, Agreement. Fetched at runtime.
Document CategoryPlatform referenceSub-classification under a type, e.g. Proof of Identity, Portfolio, Risk.
Document SpecificationTenantA named rule binding a type + categories + entity types and the compliance / signature / expiration requirements. Controls what may be uploaded.
FolderTenantHierarchical container (self-referential parentId) for organising documents.
DocumentTenantThe uploaded file plus metadata: specification, type, category, assigned entity, related entities, status, expiration.
Entity binding: every document is assigned to one primary entity (assignedEntityId) whose kind is the specification's primaryEntityType (CONTACT, HOUSEHOLD, ACCOUNT, ORGANIZATION, or PORTFOLIO) and may reference additional relatedEntities.

Specifications depend on categories. A specification references categoryIds, so the relevant document types and categories must exist before you create specifications. Resolve their IDs from the types/categories endpoints rather than hard-coding them.

Types and categories

Document types and categories are platform reference data. Resolve their IDs at runtime; do not hard-code them.

EndpointPurposeReturns
GET /api/document-types/allAll document types.AllDocumentTypesList (types[], totalCount)
GET /api/document-types/{documentTypeId}/categoriesCategories under a type.AllDocumentCategoriesList
GET /api/document-types/categories/{categoryId}A single category.DocumentCategory

Typical reference data: types Document and Agreement; categories such as Proof of Identity, Proof of Income, Proof of Address, Portfolio, Risk, and Account Administration.

Specifications

A document specification is a tenant-defined rule for one kind of document. It binds a document type and one or more categories, declares the primary and related entity types it applies to, and sets whether the document requires expiration, compliance review, and signature.

EndpointPurpose
GET /api/document-specificationsSearch specifications. Query: keyword, categoryIds, documentTypeId, sortColumn, sortOrder, page, size. Returns PagedResponse.
POST /api/document-specificationsCreate a specification. Returns 201.
GET /api/document-specifications/{id}Single specification.
GET /api/document-specifications/category/{categoryId}Specifications in a category.
PUT /api/document-specifications/{id}Update a specification.
DELETE /api/document-specifications/{id}Delete a specification. Returns 204.

POST/api/document-specifications

{
  "name":               "Cedula de identidad",
  "description":        "National ID document",
  "primaryEntityType":  "CONTACT",
  "relatedEntityTypes": ["CONTACT"],
  "requiresExpiration": true,
  "requiresCompliance": true,
  "requiresSignature":  false,
  "enabled":            true,
  "importable":         true,
  "categoryIds":        [11]
}
FieldTypeRequiredNotes
namestringrequiredMax 250 characters.
descriptionstringoptional
primaryEntityTypeenumrequiredSee EntityType.
relatedEntityTypesenum[]optionalOther entity kinds the document may reference.
requiresExpirationbooleanrequiredIf true, uploads should carry an expirationDate.
requiresCompliancebooleanrequiredEnables compliance-status tracking.
requiresSignaturebooleanrequiredEnables signature-status tracking.
enabledbooleanrequiredWhether the specification is selectable.
importablebooleanrequiredWhether documents may be imported (vs. created only).
categoryIdsinteger[]requiredAt least one category ID.
Example specification set. A tenant's locker typically defines specifications such as Cedula de identidad (Document → Proof of Identity, CONTACT, expiry + compliance), Pasaporte (Document → Proof of Identity, CONTACT), Contrato de gestion de inversiones (Agreement → Portfolio, PORTFOLIO, expiry + compliance), and Portfolio (Agreement → Portfolio, PORTFOLIO with related CONTACT and HOUSEHOLD).

Folders

Folders organise stored documents and may nest via parentId. Deleting a folder unlinks its documents rather than deleting them.

EndpointPurpose
POST /api/foldersCreate a folder (name required, optional parentId). Returns 201.
GET /api/foldersList root folders as a tree. Query: keyword, page, size.
GET /api/folders/{id}Folder detail with paginated child items (folders + documents). Query: keyword, complianceStatus, signatureStatus (comma-separated), page, size.
PUT /api/folders/{id}Rename a folder (unique within its level).
DELETE /api/folders/{id}Delete a folder; documents are unlinked, not deleted. Returns 204.
POST /api/folders/{id}/documents/{documentId}Move a document into the folder. Returns the document.

POST/api/documents

Uploads a document. Send multipart/form-data with a metadata part (JSON) and a file part (the binary). Returns 201 Created with a Document.

curl -X POST "https://api.altafid.dev.altafid.net/api/documents" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "x-tenant-uuid: YOUR_TENANT_UUID" \
  -H "x-user-id: YOUR_USER_ID" \
  -H "x-user-email: YOUR_EMAIL" \
  -H "x-user-type: TENANT_STAFF" \
  -F 'metadata={
        "documentSpecificationId": 145,
        "folderId": 43,
        "documentTypeId": 1,
        "documentCategoryId": 11,
        "assignedEntityId": "C019B9D0FD0E5772F9E8ED189895D40C4",
        "relatedEntities": [
          { "entityType": "ACCOUNT", "entityId": "A019..." }
        ],
        "name": "Cedula - Jane Doe",
        "expirationDate": "2030-12-31 23:59:59",
        "complianceStatus": "PENDING",
        "signatureStatus": "NOT_APPLY"
      };type=application/json' \
  -F 'file=@cedula.pdf'

Service integrations. Three specialised upload routes attach generated files to platform objects: POST /api/documents/report/{reportId} (report-service stores a generated PDF and gets back a documentId), POST /api/documents/portfolio/{portfolioId}/strategy/{strategyId}/proposal/{proposalId}, and POST /api/documents/portfolio/{portfolioId}/strategy/{strategyId}/trade-recommendation.

Update, move, delete

PUT/api/documents/{id} updates metadata only — name, complianceStatus, signatureStatus, relatedEntities, expirationDate. Omitted fields are unchanged.

POST/api/documents/{id} (multipart file) replaces the stored file without touching metadata.

POST/api/folders/{id}/documents/{documentId} moves a document into a folder.

DELETE/api/documents/{id} soft-deletes (204); recover via GET /api/documents/deleted then re-link.

List, filter, download

POST/api/documents/filter filters with query-param sorting (sortColumn, sortOrder, page, size).

{
  "assignedEntityIds":  ["C019B9D0FD0E5772F9E8ED189895D40C4"],
  "documentTypeId":     1,
  "complianceStatus":   ["PENDING", "APPROVED"],
  "signatureStatus":    ["SIGNED"],
  "expired":            false,
  "createdFrom":        "2026-01-01",
  "createdTo":          "2026-05-31",
  "name":               "cedula"
}
FieldTypeNotes
assignedEntityIdsstring[]Primary entity UUIDs.
relatedEntitiesobject[]{ entityType, entityId }.
documentTypeId / documentSpecificationIdintegerNarrow by type or specification.
complianceStatus / signatureStatusstring[]Status filters.
expiredbooleanPast-expiration filter.
createdFrom / createdTodateCreation date range.
namestringName search.

Download a single file with GET/api/documents/{id}/download. For several files, POST/api/documents/download/zip with a JSON array of document IDs streams an application/zip archive (204 if none match).

DocumentSpecification

id, name, description, primaryEntityType, relatedEntityTypes[], documentCategoryIds[], documentTypeId, documentType (name), importable, requiresCompliance, requiresExpiration, requiresSignature, enabled.

Folder

id, name, parentId, parentName, items[] (each a FolderItem with type = FOLDER | DOCUMENT and the corresponding folder or document), childFolderCount, documentCount.

Document type & category

DocumentType: id, name, description. DocumentCategory: id, name, description, documentType (name), documentTypeId. List wrappers carry types[], totalCount, and message.

PagedResponse

Pagination envelope: content[], page, size, totalElements, totalPages.
