Organisations API

External vendor

Organisations support classic CRUD operations plus the same v2 filtering model used by other major entity endpoints.

Read endpoints

EndpointPurpose
GET /api/organisationsPaginated list for the current tenant.
GET /api/organisations/{organisationUuid}Get one organisation.
GET /api/organisations/search?searchTerm=Search by legal name, company name, or identification ID.
GET /api/organisations/{organisationUuid}/householdsList linked households.

Classic list and search endpoints accept includeCustomFields=true when you need the expanded response.

Create and update

POST/api/organisations

{
  "legalCompanyName": "Acme Advisory LLC",
  "companyName": "Acme Advisory",
  "industry": "FINANCIAL_SERVICES",
  "identificationType": "EIN",
  "identificationId": "12-3456789",
  "createdByTenantStaffUuid": "TS019C06688AA57248B9EA5325DBDFC48E",
  "preferredPhoneNumber": {
    "countryCode": "+1",
    "number": "5557771234"
  },
  "administratorContactUuids": [],
  "associatedContactUuids": []
}
Required on createNotes
legalCompanyNameRequired string.
industryOrganisation industry enum.
identificationTypeIdentity type enum.
identificationIdTenant-scoped duplicate checks apply.
createdByTenantStaffUuidRequired creator reference.

PUT/api/organisations/{organisationUuid} accepts optional replacements for phone numbers, addresses, administrators, and associated contacts.

{
  "companyName": "Acme Advisory Group",
  "isActive": true,
  "lockerId": "LOCKER-1001",
  "administratorContactUuids": ["C019B9D0FD0E5772F9E8ED189895D40C4"]
}

DELETE/api/organisations/{organisationUuid} returns 204 on success.

V2 filtering

EndpointPurpose
POST /api/v2/organisationsComplex filtering, sorting, and pagination.
POST /api/v2/organisations/filter-valuesDistinct values for a field, optionally against base filters.
 
{
  "filters": [
    { "field": "isActive", "operator": "equals", "value": "true" }
  ],
  "page": 0,
  "size": 20,
  "sort": ["legalCompanyName,asc"]
}
