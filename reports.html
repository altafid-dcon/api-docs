Reports API

External integration

The Reports API generates account and household reports. A report type is a reusable template made of ordered processing steps; a report is one generated instance for a specific account or household; a subscription is a standing order that produces reports automatically on a schedule.

Base URL: https://api.altafid.dev.altafid.net
All requests require standard auth headers — see Authentication and Global Headers.
Generation is asynchronous. Creating a report returns immediately with status PENDING. The report then moves to PROCESSING and finally COMPLETED (or FAILED). When complete, fileId references the generated PDF in the Documents service, where it is downloaded.

Authentication and headers

HeaderRequiredNotes
Authorization: Bearer <token>requiredAccess token.
x-tenant-uuidrequiredTenant context.
x-user-idrequiredCurrent user UUID.
x-user-emailrequiredCurrent user email.
x-user-typerequiredTENANT_STAFF or PARTNER_STAFF.
x-partner-uuidoptionalRequired for partner staff.

Object model

ConceptWhat it is
Report TypeA tenant template: a code, target entity type (ACCOUNT or HOUSEHOLD), periodic flags, and an ordered list of steps.
Report Type StepOne ordered processing step (stepOrder + a ReportStepCode) that fetches a slice of data — e.g. HOUSEHOLD_BALANCE_CHART, ACCOUNT_ASSET_ALLOCATION.
ReportOne generated instance for a given entity and period, with a lifecycle status and a resulting fileId.
SubscriptionA standing order, one per account/household, holding currency + language defaults for reports generated automatically on schedule.
Reporting is on accounts and households. Every report and subscription targets an entityType of ACCOUNT or HOUSEHOLD by entityId, drawing data from the Accounts service.

Report type steps

Steps define, in order, what data a report type collects. Each step is a ReportStepCode mapped to a data-fetch operation against the accounts/workstation services.

EndpointPurpose
GET /api/report-types/{id}/stepsList the steps configured for a report type.
POST /api/report-types/{id}/stepsAdd a step (code, stepOrder, optional resultKey). Returns 201.
DELETE /api/report-types/{id}/steps/{stepId}Remove a step. Returns 204.

Status and download

Poll GET/api/reports/{id} until status is COMPLETED. The response then carries fileId — the generated PDF's reference in the Documents service, where the file is downloaded. If generation fails, status is FAILED and errorMessage explains why.

POST/api/reports/filter searches reports. Sortable on id, name, entityType, status, periodStart, periodEnd, requestedAt, requestedBy, createdAt.

{
  "reportTypeIds":   [5],
  "entityType":      ["HOUSEHOLD"],
  "entityIds":       ["H019B9D0FD0E5772F9E8ED189895D40C4"],
  "status":          ["COMPLETED", "PROCESSING"],
  "name":            "annual",
  "periodStartFrom": "2026-01-01",
  "periodStartTo":   "2026-12-31",
  "requestedBy":     ["PS0199CA8F08C270B5A5B0DFD3B565B332"]
}

Subscriptions

A subscription is a standing order to generate reports automatically for an account or household. There is one subscription per entity; it holds the default currency and language. Subscriptions are seeded for all active entities and then tuned.

EndpointPurpose
POST /api/reports/subscriptions/setupScan all active accounts/households and upsert any missing subscriptions. Idempotent. TENANT_ADMIN only.
POST /api/reports/subscriptions/filterList subscriptions by entityType, entityIds, entityName. Paginated.
PUT /api/reports/subscriptions/{id}Update currency and/or language.
PUT /api/reports/subscriptions/bulkUpdate many subscriptions in one call.
 
[
  { "subscriptionId": 1, "currency": "USD", "language": "en" },
  { "subscriptionId": 2, "currency": "EUR", "language": "es" }
]

Scheduler

POST/api/reports/scheduler/trigger runs the periodic generation logic for a window without waiting for the cron — intended for testing. Query params periodStart and periodEnd (yyyy-MM-dd) are required.

curl -X POST "https://api.altafid.dev.altafid.net/api/reports/scheduler/trigger?periodStart=2026-01-01&periodEnd=2026-01-31" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "x-tenant-uuid: YOUR_TENANT_UUID" \
  -H "x-user-id: YOUR_USER_ID" \
  -H "x-user-email: YOUR_EMAIL" \
  -H "x-user-type: TENANT_STAFF"

ReportType & Step

ReportType: id, tenantUuid, code, name, entityType, description, isPeriodic, isYearlyEnabled, isQuarterlyEnabled, templateId, createdAt, updatedAt. Step: id, reportTypeId, stepOrder, code (ReportStepCode), resultKey.

Subscription

id, tenantUuid, entityId, entityName, entityType, currency, language, createdAt, updatedAt.

PagedResponse

Pagination envelope: content[], page, size, totalElements, totalPages.
